Introduction on the Semantic Web

The best definition of this concept may be "The Semantic Web is an extension of the current web in which information is given well-defined meaning, better enabling computers and people to work in cooperation"(presented in the Scientific Article "The Sematic Web" by Berners Lee). The inventor thought that the Web can intuit more the users" needs and how to perform them.

But what exactly is the Semantic Web?

Every language is characterized by Syntax and Semantic. Syntax concerns with grammar, whilst Semantics studies the meaning.

  • The syntaxes can be different, but they have the same semantic: (e.g. x ++ / x += 1/ x = x+1)
Web is a collection of interlinked documents, that can be easily accessed by users via INTERNET.

So, the main difference between Semantic Web technology and other technologies involving data (e.g. relational databases) is that Semantic Web studies the meaning and not the structure of data. This leads to a different manner of storing, displaying information and querying, which is beneficial for the applications that deal with a large amount of data.

The Semantic Web concerns with concepts, recognizing persons, locations, movies etc. and understanding the connections/ relationships between them. It involves basically three technical principles:

  • RDF (Resource Description Framework): a standard model for data interchange on the Web (data modelling language for Semantic Web)
  • SPARQL (SPARQL Protocol and RDF Query Language) : query language for the Semantic Web, designed to query data among the systems
  • OWL (Web Ontology Language) : the schema language of the Semantic Web designed to represent complex information about groups of things, relationship between them

RDF:

RDF is designed for describing the resources through Web and it can also recognize things with the help of URIs (Uniform Resource Identifiers).
For describing things, RDF uses simple statements (Triples: Subject - Predicate- Object). Though, the most promising application of RDF is not in encoding information about Web resources, but modelling a relational database.

RDF is based on four rules:

  • A fact is represented by triples
  • Subjects, predicates and objects are names for entities or resources (referring to the use of RDF to metadata in web resources)
  • Names are expressed by URIs, that can have two main applications:
    • URNs (Uniform Resource Names) that are the URIs used for naming not only Web objects, but also abstract ones. For example, a person can have one URI used for referring to that person in the ontologies.
    • URLs (Uniform Resource Locators) that are the URIs used for setting out the location of the objects/things. Likewise URIs start with a protocol identifier(indicates the protocol to be used for obtaining the resource), URLs are composed of a protocol, http used to access the source at the specified location.
  • Objects can be expressed as strings, also called literal values

SPARQL:

In other words, SPARQL allows users to query data from databases that can be mapped to RDF. It matches the patterns in the query to the triples in the dataset, to find the solution for the variables (previously mentioned subject/predicate/object).

There are four types of queries:

  • ASK if there exists at least one match of the query pattern in the RDF
  • SELECT a certain number of matches and convert them into a tabular form
  • CONSTRUCT an RDF graph by substituting the variables in those matches in a set of triples
  • DESCRIBE the matches found by constructing an RDF graph

OWL:

OWL is a computational logic-based language, which expresses information in a machine-readable way such that it is possible to verify consistency or to make implicit information explicit.
